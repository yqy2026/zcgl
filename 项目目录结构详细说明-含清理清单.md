## ⚠️ 需要立即清理的文件清单

### 1. Python 缓存文件（应删除并添加到 .gitignore）

- **`backend/__pycache__/`**（整个目录）
  - **类型**：生成物（Python 字节码缓存）
  - **问题**：Python 自动生成的字节码缓存，不应提交到版本控制
  - **操作**：删除整个目录，在 `.gitignore` 中添加 `__pycache__/`

### 2. SQLite 临时文件（应删除并添加到 .gitignore）

- **`backend/test.db-shm`**
  - **类型**：生成物（SQLite 共享内存文件）
  - **问题**：SQLite 自动生成的临时文件
  - **操作**：删除，在 `.gitignore` 中添加 `*.db-shm`

- **`backend/test.db-wal`**
  - **类型**：生成物（SQLite WAL 文件）
  - **问题**：SQLite 自动生成的临时文件
  - **操作**：删除，在 `.gitignore` 中添加 `*.db-wal`

- **`backend/src/data/defect_tracking.db`**
  - **类型**：生成物（测试数据库文件）
  - **问题**：临时测试数据，不应提交
  - **操作**：删除或移动到 `backend/test-data/` 目录

### 3. 测试覆盖率报告文件（应删除并添加到 .gitignore）

- **`backend/htmlcov/`**（整个目录，包含 100+ 个 HTML 文件）
  - **类型**：生成物（HTML 覆盖率报告）
  - **问题**：pytest 自动生成的 HTML 报告，占用空间大
  - **操作**：删除整个目录，在 `.gitignore` 中添加 `htmlcov/`

- **`backend/coverage.json`**, **`backend/coverage.xml`**
  - **类型**：生成物（覆盖率报告）
  - **问题**：测试生成的报告文件，不应提交
  - **操作**：删除，在 `.gitignore` 中添加 `coverage.json`, `coverage.xml`

- **`backend/junit-report.xml`**
  - **类型**：生成物（JUnit 测试报告）
  - **问题**：测试生成的报告文件，不应提交
  - **操作**：删除，在 `.gitignore` 中添加 `junit-report.xml`

- **根目录的测试报告文件**：
  - **`coverage.json`**, **`coverage.xml`**, **`junit-report.xml`**
  - **类型**：生成物
  - **问题**：不应在根目录，应该在 backend 目录下
  - **操作**：删除

- **`htmlcov/`**（根目录，如果存在）
  - **类型**：生成物
  - **问题**：不应在根目录
  - **操作**：删除

### 4. 日志文件（应删除并添加到 .gitignore）

- **`backend/logs/app.log`**, **`backend/logs/requests.log`**, **`backend/logs/security.log`**
  - **类型**：生成物（运行时日志）
  - **问题**：应用运行时生成的日志文件，会不断增长
  - **操作**：删除，在 `.gitignore` 中添加 `logs/` 或 `*.log`

- **`logs/app.log`**, **`logs/requests.log`**, **`logs/security.log`**（根目录）
  - **类型**：生成物（运行时日志）
  - **问题**：不应在根目录
  - **操作**：删除

### 5. 备份文件（过多，需清理）

- **`backend/backups/`** 目录
  - **当前状态**：包含 **40+ 个数据库备份文件**（`backup_*.db`）
  - **类型**：生成物（数据库备份）
  - **问题**：备份文件过多，占用大量空间，不应提交到版本控制
  - **操作**：
    1. 只保留最近 10-20 个备份文件
    2. 删除旧的备份文件
    3. 在 `.gitignore` 中添加 `backups/` 目录

- **`backups/`** 目录（根目录）
  - **当前状态**：包含多个备份文件
  - **类型**：生成物（数据库备份）
  - **问题**：不应在根目录，备份不应提交
  - **操作**：删除或只保留最近的几个备份

### 6. 重复的临时分析脚本（应删除或归档）

- **`backend/find_all_quick_wins.py`**
  - **类型**：临时分析脚本
  - **问题**：与 `find_schema_quick_wins.py` 功能重复

- **`backend/check_cache_coverage.py`**

  - **问题**：一次性使用的覆盖率分析脚本
  - **操作**：删除或移动到 `backend/scripts/archive/`

### 7. 其他临时文件

  - **问题**：Ruff 自动生成的报告，不应提交

## 总结

本项目存在大量需要清理的临时文件、缓存文件和生成物，主要集中在：

1. **测试报告和覆盖率文件**（`htmlcov/`, `coverage.*`, `junit-report.xml`）
2. **备份文件**（`backups/` 目录下 40+ 个文件）
3. **Python 缓存**（`__pycache__/`）
4. **SQLite 临时文件**（`*.db-shm`, `*.db-wal`）
5. **日志文件**（`logs/*.log`）
6. **重复的临时脚本**（多个 `find_*.py` 脚本）

建议使用 `scripts/cleanup_project.py` 脚本进行清理，并在 `.gitignore` 中添加相应的忽略规则，防止将来再次提交这些生成物。
