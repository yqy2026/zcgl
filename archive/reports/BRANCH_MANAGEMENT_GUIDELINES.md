## 🎯 目标与原则

### 核心目标
- **代码质量**: 确保代码库保持高质量和一致性
- **开发效率**: 优化团队协作和开发流程
- **风险控制**: 降低开发过程中的风险和问题
- **可追溯性**: 建立清晰的变更历史和审计轨迹

### 核心原则
- **🔐 质量优先**: 代码质量优于开发速度
- **📋 文档同步**: 代码变更必须有相应文档更新
- **🧪 测试覆盖**: 功能必须有充分的测试验证
- **🛡️ 安全第一**: 安全性是所有变更的基础要求

## 🌿 分支类型与命名规范

### 分支类型定义

| 分支类型 | 命名格式 | 用途 | 生命周期 |
|---------|----------|------|----------|
| **主分支** | `main` | 生产环境代码 | 永久 |
| **开发分支** | `develop` | 开发环境代码 | 永久 |
| **功能分支** | `feature/功能描述` | 新功能开发 | 功能完成后 |
| **热修复分支** | `hotfix/问题描述` | 紧急修复 | 修复后合并 |
| **发布分支** | `release/版本号` | 发布准备 | 发布完成后 |
| **实验分支** | `experiment/实验名称` | 实验性功能 | 实验后评估 |

### 命名规范详解

#### ✅ 正确示例
```
main                           # 主分支
develop                        # 开发分支
feature/user-authentication   # 用户认证功能
feature/pdf-ocr-enhancement   # PDF OCR增强功能
feature/system-monitoring      # 系统监控功能
hotfix/login-bug-fix          # 登录bug紧急修复
hotfix/payment-security-fix    # 支付安全紧急修复
release/v1.2.0               # 版本1.2.0发布
release/v1.2.1               # 版本1.2.1发布
experiment/ai-enhancement       # AI功能实验
```

#### ❌ 错误示例
```
master                        # 不使用master，使用main
dev                          # 不使用dev，使用develop
feature                      # 名称太简短
bugfix                       # 使用hotfix前缀
feature/user_auth            # 使用连字符分隔
feature/PDF-Import           # 避免大写字母
release/v1.2.0-hotfix        # 不要混合分支类型
```

## 🔄 分支生命周期管理

### 分支创建流程

#### 1. 功能分支创建
```bash
# 从最新的develop分支创建
git checkout develop
git pull origin develop

# 创建功能分支
git checkout -b feature/new-feature-name

# 推送到远程
git push -u origin feature/new-feature-name
```

#### 2. 分支切换
```bash
# 切换到目标分支
git checkout target-branch

# 如果是远程分支不存在
git fetch origin
git checkout -b target-branch origin/target-branch
```

### 分支合并流程

#### 1. 合并到develop分支
```bash
# 切换到develop分支
git checkout develop
git pull origin develop

# 合并功能分支
git merge feature/feature-name --no-ff

# 解决冲突（如有）
# 手动解决冲突后...
git add .
git commit -m "Merge feature/feature-name into develop"

# 推送更新
git push origin develop
```

#### 2. 合并到main分支
```bash
# 切换到main分支
git checkout main
git pull origin main

# 合并develop分支
git merge develop --no-ff

# 创建版本标签
git tag -a v1.2.0 -m "Release version 1.2.0"

# 推送标签
git push origin main --tags
```

### 分支清理流程

#### 1. 已合并分支清理
```bash
# 删除本地分支
git branch -d feature/completed-feature

# 删除远程分支
git push origin --delete feature/completed-feature
```

#### 2. 实验失败分支清理
```bash
# 确认分支不再需要
git checkout main

# 删除本地分支
git branch -D feature/abandoned-feature

# 删除远程分支
git push origin --delete feature/abandoned-feature
```

## 🚨 应急处理流程

### 问题处理流程

#### 1. CI检查失败
1. **立即通知**: 相关人员收到失败通知
2. **问题定位**: 快速定位失败原因
3. **紧急修复**: 1小时内提供修复方案
4. **重新验证**: 修复后重新触发检查
5. **根因分析**: 分析根本原因并制定改进措施

#### 2. 合并冲突处理
1. **立即通知**: 避免冲突扩大化
2. **责任分配**: 指定冲突解决负责人
3. **协同解决**: 团队协作解决冲突
4. **质量检查**: 解决后进行质量检查
5. **完成合并**: 验证后完成合并

#### 3. 生产问题回滚
1. **立即隔离**: 快速隔离问题功能
2. **评估影响**: 评估问题影响范围
3. **回滚决策**: 决定是否需要回滚
4. **执行回滚**: 执行回滚操作
5. **问题修复**: 修复问题并重新部署
