# OCR配置优化报告

## 优化目标
针对022号租赁合同扫描件OCR文字识别准确率问题进行系统性优化，提高PDF智能导入功能的处理质量。

## 原始配置问题分析

### 1. Tesseract配置过于简单
- **原始配置**: `--psm 6 --oem 3`
- **问题**: 单一页面分割模式无法适应不同文档结构
- **影响**: 对合同文档的关键字段识别率较低

### 2. 图像预处理不够优化
- **原始处理**: 简单的二值化（固定阈值150）+ 基础对比度增强
- **问题**:
  - 固定阈值不适应不同质量的扫描件
  - 缺少倾斜校正
  - 降噪处理过于基础
- **影响**: 图像质量改善有限，OCR准确率提升不明显

### 3. 缺少质量评估机制
- **原始状态**: 无置信度评估
- **问题**: 无法选择最佳OCR结果
- **影响**: 次优结果被直接使用，影响最终数据质量

## 优化实施方案

### 1. 多重Tesseract配置机制

#### 优化前:
```python
self.tesseract_config = {
    'lang': 'chi_sim+eng',
    'config': '--psm 6 --oem 3'
}
```

#### 优化后:
```python
self.tesseract_config = {
    'lang': 'chi_sim+eng',
    'config': '--psm 4 --oem 3 -c tessedit_char_whitelist=0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ一二三四五六七八九十百千万零元月日年号楼号房号平方米甲方乙方合同租赁期限地址租金押金：：，。、；；（）【】',
    'alternative_configs': [
        '--psm 6 --oem 3',  # 原配置作为备用
        '--psm 3 --oem 3',  # 完全自动
        '--psm 1 --oem 3',  # 带OSD的自动页面分割
        '--psm 11 --oem 3', # 稀疏文字
        '--psm 12 --oem 3'  # 带OSD的稀疏文字
    ]
}
```

#### 优化要点:
- **主配置**: PSM 4（单列文本）更适合合同文档
- **字符白名单**: 限制识别范围，减少错误字符
- **备用配置**: 5种不同PSM模式应对各种文档布局
- **自动选择**: 根据置信度自动选择最佳配置

### 2. 高级图像预处理算法

#### 优化前:
- 固定阈值二值化
- 基础对比度增强(2.0倍)
- 基础锐化(1.5倍)

#### 优化后:
- **自适应对比度增强**: 降低强度至1.5倍，避免过度处理
- **适度锐化**: 降低强度至1.2倍，保持自然度
- **智能降噪**: 双边滤波 + 形态学操作
- **倾斜校正**: 霍夫变换检测 + 自动旋转校正
- **自适应阈值**: Otsu算法或统计均值动态确定阈值
- **直方图均衡化**: CLAHE自适应对比度增强

#### 工作流程:

2. **置信度评估**: 计算识别结果质量评分
3. **备用配置尝试**: 如果置信度<0.3，尝试5种备用配置
4. **最优选择**: 选择置信度最高的结果
5. **详细日志**: 记录配置选择过程和评分
