# 资产管理 PRD

## ✅ Status
**当前状态**: Draft (2026-02-04)

## 背景与目标
统一资产档案管理流程，提升数据准确性、可检索性与跨模块协同效率，支持基础统计分析。

## 目标范围
- 资产档案管理（字段维护、查询、导入/导出、批量操作）
- 资产检索/筛选、附件管理与自定义字段
- 资产与项目/权属方/合同/产权证的关联能力
- 资产统计与分析的基础数据能力

## 非目标
- 资产估值模型
- 高级 BI 报表体系

## 关键需求
### 1) 核心对象与关系
- 资产是核心实体，可选关联项目与权属方
- 合同/产权证必须关联资产（资产可选不关联）
- 资产支持附件（PDF）与自定义字段
- 资产变更需要保留历史记录（审计字段）

### 2) 枚举字段与业务含义
- **确权状态**: 已确权 / 部分确权 / 未确权 / 其它
- **使用状态**: 出租 / 闲置 / 自用 / 公房（出租） / 公房（闲置） / 其它
- **物业性质**: 经营类 / 公房 / 自用 / 其它
- **租户类型**: 企业 / 个人 / 其它

### 3) 数据规则与校验
- **资产 `property_name` 全局唯一**（创建/更新必须避免重名）
- 导入去重认可 `property_name + address` 逻辑
- 计算字段 `unrented_area` / `occupancy_rate` 不落库，实时计算
- 面积一致性：`rented_area ≤ rentable_area`
- 以 `ownership_id` 为唯一来源；`ownership_entity` 仅动态读取（不落库）
- 导入/写入如携带 `ownership_entity`，仅用于匹配 `ownership_id`，不作为持久化字段

### 4) 业务流程
- 资产 CRUD + 列表查询（分页/搜索/筛选/排序）
- 批量更新/删除
- 附件上传/下载/删除（PDF、大小限制）

### 5) 删除/关联约束
- 资产删除时若已关联合同或产权证，必须阻止删除

### 6) 权限与数据范围（预留）
- 当前按角色权限控制
- 未来按**角色**配置可见范围（权属方/经营方关注点不同）
- 跨组织共享规则待后续明确

### 7) 当前未落地但已明确“暂不做”
- 暂无状态驱动必填规则（如“已出租必须填写租户/合同”）
- 暂不需要审批流/审核状态（当前仅有 `audit_notes`）

## 历史数据迁移策略（ownership_entity → ownership_id）
- 对历史资产仅存 `ownership_entity` 的记录，按权属方名称匹配 `ownerships.name` 回填 `ownership_id`
- 回填完成后不再保留 `ownership_entity` 存储字段（统一改为动态读取）
- 若名称无法匹配或 `ownership_id` 指向不存在记录，记录异常并跳过

## 验收标准
- 资产 CRUD 可用且符合唯一性约束
- 关键筛选、导入/导出与批量操作可用
- 关联关系正确保存，删除校验生效
