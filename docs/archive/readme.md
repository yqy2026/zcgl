# 归档目录说明

`docs/archive/` 用于存放历史草稿、阶段性复盘、过期计划与不再作为当前基线的文档。

## 规则

- 归档文档不作为当前需求或实现基线。
- 新增归档时无需修改此文件，按子目录归类存放即可。
- 引用归档内容时应同时给出现行基线文档链接。

## 子目录说明

| 子目录 | 内容 |
|--------|------|
| `evidence/` | 阶段性门禁证据、发布证据和本地验证快照 |
| `guides/` | 已从当前入口移除的历史指南和长版设计快照 |
| `backend-plans/` | 后端已完结技术方案（从 `plans/` 归档） |
| `requirements-decisions/` | 需求访谈、评审、As-Built、迁移过程归档 |
| `reviews/` | 方案或代码审阅记录归档 |
