# Claude Code 自定义命令使用指南

本项目为 Claude Code 添加了自定义斜杠命令，用于快速执行常见任务。

## 📋 可用命令

| 命令 | 功能 | 使用场景 | 执行时间 |
|------|------|---------|---------|
| `/diagnose` | 前端运行时诊断 | 检查前端错误、性能、网络请求 | 1-2 分钟 |
| `/quick-dev` | 快速开发检查 | 日常开发前的快速验证 | 30 秒 |
| `/check-all` | 完整项目检查 | PR 提交前、版本发布前 | 10-15 分钟 |
| `/clean` | 项目文件清理 | 清理临时文件、缓存、测试产物 | 1-2 分钟 |

## 🚀 快速开始

### 1. 前端诊断命令

**何时使用**:
- 开发新功能后，检查是否有运行时错误
- 修改 API 相关代码后，验证请求是否正常
- 提交代码前，确保前端质量
- 页面出现问题时，快速诊断

**使用方法**:
```
/diagnose
```

## 💡 使用建议

### 每日开发流程

```
1. 早上开始开发
   /quick-dev  (30秒)

2. 开发过程中
   (保存文件，IDE 自动检查)

3. 完成功能后
   /diagnose  (检查前端运行时)

4. 提交代码前
   /check-all  (完整检查)
```

### 故障排查流程

```
问题: 前端页面报错
  ↓
/diagnose  (诊断运行时错误)
  ↓
查看 HTML 报告
  ↓
定位错误文件和行号
  ↓
修复错误
  ↓
/quick-dev  (快速验证)
```

### PR 提交流程

```
1. 功能开发完成
   /check-all  (完整检查)

2. 修复所有问题
   /check-all  (再次验证)

3. 提交 PR
   (CI 会自动运行 /check-all)

4. Review 通过
   (合并代码)
```

## 🔧 自定义命令

### 创建新命令

1. 在 `.claude/commands/` 创建新文件
2. 使用 Markdown 格式编写
3. 文件名即为命令名（如 `my-command.md`）
4. 使用 `/my-command` 调用

### 命令模板

```markdown
# 命令名称

简短描述命令的功能。

## 使用场景

- 场景 1
- 场景 2

## 执行步骤

### 步骤 1: ...
```bash
# 命令
```

### 步骤 2: ...
```bash
# 命令
```

## 预期结果

- 结果 1
- 结果 2

## 常见问题

### 问题 1
**解决方案**: ...

---

**执行时间**: X 分钟
**依赖**: ...
```
