## 根本原因分析

### 1. 正则表达式边界检测问题
**问题描述**: 多个字段的正则表达式没有正确设置边界条件，导致过度捕获文本。

**具体表现**:
- `landlord_legal_rep`字段：`r"法定代表人[：:]([^，,。\n]+)"` 没有设置正确的结束边界
- `landlord_contact`字段：`r"出租方.*?联系人[：:]([^，,。\n]+)"` 没有正确匹配联系人
- 多个字段使用了过于宽泛的字符类`[^，,。\n]+`

### 2. 字段顺序依赖问题
**问题描述**: 提取器没有考虑到合同文本中字段出现的顺序，导致后续字段被前面字段的模式捕获。

### 3. 缺失关键字段模式
**问题描述**: 以下字段在提取规则中完全缺失：
- 月租金提取模式
- 保证金提取模式
- 签订日期提取模式

## 修复建议

### 1. 立即修复 - 高优先级

#### 修复出租方信息提取
```python
# 更新出租方字段的正则表达式
"landlord_name": [
    r"出租方[（(]甲方[）)][：:]([^，,。\n]+?)(?=法定代表人|统一社会|联系地址|联系人|联系电话)",
    r"甲方[：:]([^，,。\n]+?)(?=法定代表人|统一社会|联系地址|联系人|联系电话)"
],
"landlord_legal_rep": [
    r"法定代表人[：:]([^，,。\n]+?)(?=统一社会|联系地址|联系人|联系电话)",
    r"出租方.*?法定代表人[：:]([^，,。\n]+?)(?=统一社会|联系地址|联系人|联系电话)"
],
"landlord_contact": [
    r"联系人[：:]([^，,。\n]+?)(?=联系电话|电子邮箱)",
    r"出租方.*?联系人[：:]([^，,。\n]+?)(?=联系电话|电子邮箱)"

    r"联系电话[：:]([0-9-]+?)(?=[^\d-]|$)",
```

### 2. 优化边界检测 - 中优先级

#### 改进所有字段的边界检测
- 使用非贪婪匹配：`+?` 代替 `+`
- 添加正确的向前查找：`(?=...)`
- 增加停止词：如"身份证号码"、"通讯地址"、"联系电话"等

#### 优化字段处理顺序
- 按照合同中字段出现的自然顺序处理
- 避免后续字段被前面字段捕获
