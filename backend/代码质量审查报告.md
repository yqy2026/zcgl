## 🚨 严重安全问题 (Critical Security Issues)

### 1. [CVE-2025-001] JWT 密钥硬编码风险
**文件**: `backend/src/core/config.py:46`  
**风险等级**: 🔴 极高  
**问题描述**: 
```python
SECRET_KEY: str = Field(
    default="your-secret-key-change-in-production",  # 硬编码的默认密钥
    json_schema_extra={"env": "SECRET_KEY"},
)
```
**影响**: 
- 默认密钥可能被攻击者轻易猜测
- 生产环境如果未修改环境变量，所有JWT令牌可被伪造
- 可能导致未授权访问和权限提升

**修复建议**:
```python
SECRET_KEY: str = Field(
    ...,
    json_schema_extra={"env": "SECRET_KEY"},
)
# 强制要求环境变量，不提供默认值
```

### 2. [CVE-2025-002] SQL注入风险
**文件**: `backend/src/api/v1/analytics.py:108`  
**风险等级**: 🔴 高  
**问题描述**: 直接使用用户输入构建查询
```python
query = db.query(Asset).filter(Asset.data_status == DataStatus.NORMAL.value)
# 缺少参数化查询保护
```
**修复建议**: 使用参数化查询或ORM过滤方法

### 3. [CVE-2025-003] 密码历史记录存储不当
**文件**: `backend/src/services/core/auth_service.py:150-160`  
**风险等级**: 🟠 中高  
**问题描述**: 
```python
user.password_history = json.dumps(password_history)
# 明文存储密码哈希历史，可能被用于彩虹表攻击
```
**修复建议**: 对密码历史进行加密存储

### 4. [CVE-2025-004] 文件上传路径遍历漏洞
**文件**: `backend/src/api/v1/assets.py:200-250`  
**风险等级**: 🟠 中高  
**问题描述**: 
```python
file_path = os.path.join(UPLOAD_DIR, filename)
# 未对filename进行充分验证，可能导致路径遍历
```
**修复建议**: 使用安全的文件名处理函数

### 5. [CVE-2025-005] 敏感信息日志泄露
**文件**: `backend/src/middleware/security_middleware.py:150-160`  
**风险等级**: 🟠 中  
**问题描述**: 
```python
logger.error(f"Security middleware error: {error_message}")
# 可能在日志中泄露敏感信息
```
**修复建议**: 对日志输出进行脱敏处理

### 6. [CVE-2025-006] CORS配置过于宽松
**文件**: `backend/src/middleware/security_middleware.py:380-390`  
**风险等级**: 🟠 中  
**问题描述**: 
```python

    response.headers["Access-Control-Allow-Origin"] = origin
# 允许任何localhost子域名，可能被滥用
```
**修复建议**: 严格限制允许的源

**文件**: `backend/src/database.py:85-95`  
**问题**: get_db()函数异常处理不完整

        db = SessionLocal()
        yield db
    except Exception as e:
        logger.error(f"Database session error: {e}")

        if db is not None:
            db.close()  # 应该在finally块中确保关闭
