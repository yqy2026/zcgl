#### 创建的配置

**生产环境 Nginx** - `nginx/production_nginx.conf`
- **功能**: 企业级负载均衡和反向代理配置
- **特性**:
  - 负载均衡：least_conn算法，健康检查
  - SSL/TLS终止：现代加密配置
  - 压缩优化：Gzip + Brotli 双重压缩
  - 缓存策略：API缓存 + 静态文件缓存
  - 速率限制：API + 上传独立限速
  - 安全头：完整的安全HTTP头
  - 请求优化：连接池、缓冲区优化

**性能优化亮点**:
```nginx
# 负载均衡配置
upstream fastapi_backend {
    least_conn;
    server backend1:8002 weight=3 max_fails=3 fail_timeout=30s;
    server backend2:8002 weight=2 max_fails=3 fail_timeout=30s;
    server backend3:8002 weight=1 max_fails=3 fail_timeout=30s backup;
}

# 智能缓存配置
proxy_cache_path /var/cache/nginx/proxy_cache levels=1:2 keys_zone=api_cache:100m inactive=60m;
proxy_cache_key $scheme$proxy_method$host$request_uri;
proxy_cache_valid 200 302 5m;
```

## 使用指南

### 快速部署
```bash
# 1. 克隆代码
git clone https://github.com/your-org/asset-management.git
cd asset-management

# 2. 配置环境
cp .env.production.example .env.production
# 编辑 .env.production 配置文件

# 3. 一键部署
chmod +x scripts/deploy.sh
./scripts/deploy.sh production v1.0.0

# 4. 验证部署
curl -f http://your-domain.com/api/v1/health
```

### 监控访问
- **应用访问**: https://your-domain.com

- **监控面板**: https://your-domain.com:3000
- **指标查询**: https://your-domain.com:9090
- **链路追踪**: https://your-domain.com:16686

```bash

docker-compose -f docker/production/docker-compose.yml ps

# 查看服务日志
docker-compose -f docker/production/docker-compose.yml logs -f

# 扩容服务实例
docker-compose -f docker/production/docker-compose.yml up -d --scale backend=5

# 更新服务
docker-compose -f docker/production/docker-compose.yml pull
docker-compose -f docker/production/docker-compose.yml up -d
```
